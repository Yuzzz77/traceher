# 常见任务流程

## 做一个作品集网页

步骤：

1. 明确主题：这个网页是展示个人、作品、产品还是交互效果。
2. 确定第一屏：打开页面后用户第一眼看到什么。
3. 实现基础结构：HTML 内容、CSS 布局、JS 交互。
4. 增加亮点：粒子、动画、滚动、3D、数据可视化等。
5. 检查体验：文字是否看得清，按钮是否能点，移动端是否错乱。
6. 总结亮点：写出可以放到简历里的项目描述。

## 做粒子交互系统

步骤：

1. 先确定粒子表现：星空、流体、文字粒子、鼠标吸引、爆炸、连线等。
2. 选择技术：简单 2D 用 Canvas，复杂 3D 用 Three.js。
3. 先做可运行版本：画出粒子并让它们动起来。
4. 再加交互：鼠标移动、点击、拖拽、参数面板。
5. 最后加展示价值：标题、说明、截图角度、作品集描述。

## 修 bug

步骤：

1. 先复现问题。
2. 找到相关代码。
3. 解释问题原因。
4. 做最小修改。
5. 再次验证。

不要没复现就盲改。

## 优化视觉效果

步骤：

1. 先判断当前页面的问题：乱、空、暗、土、没重点、交互弱。
2. 保留原有核心功能。
3. 优先优化布局、字体、颜色、间距、动效。
4. 不要为了装饰堆太多元素。
5. 最后检查文字有没有重叠、按钮有没有变形。

## 生成简历项目描述

输出格式：

```text
项目名称：
项目简介：
技术栈：
我负责的内容：
项目亮点：
可以继续优化的方向：
```

描述要真实，不要夸大成企业级项目。
