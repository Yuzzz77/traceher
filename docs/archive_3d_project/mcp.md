# MCP 使用建议

MCP 可以理解成“大模型的工具接口”。

普通聊天模型只能回答问题；接入 MCP 后，模型可以使用外部工具，例如：

- 读写本地文件
- 操作浏览器
- 查询数据库
- 访问设计稿
- 运行命令
- 管理 GitHub issue

## 对当前用户最有用的 MCP

优先级从高到低：

1. filesystem：让模型读取和修改本地项目文件。
2. browser / playwright：让模型打开网页、截图、检查页面效果。
3. git：让模型查看代码改动和历史记录。
4. github：以后把作品上传到 GitHub 后，用来管理仓库。
5. fetch / web search：查官方文档和资料。

## 新手推荐配置

先不要一次装太多 MCP。建议顺序：

1. 先配置 filesystem。
2. 再配置 browser 或 playwright。
3. 项目开始上传 GitHub 后，再配置 github。
4. 等需要数据库或自动化时，再加其他 MCP。

## 使用原则

每次让模型使用 MCP 前，先说清楚目标：

```text
请打开当前 HTML 文件，检查页面有没有显示异常，并告诉我哪里需要改。
```

不要只说：

```text
帮我看看。
```

目标越清楚，工具用得越准。
