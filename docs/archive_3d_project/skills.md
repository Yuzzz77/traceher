# Skill 使用建议

Skill 可以理解成“大模型的专项能力说明书”。

Agent 是“角色和工作方式”。

MCP 是“工具接口”。

Skill 是“某类任务的固定做法”。

例如：

- 粒子交互 Skill：专门指导模型怎么做粒子系统。
- 作品集页面 Skill：专门指导模型怎么做适合求职展示的页面。
- 简历项目描述 Skill：专门指导模型怎么把项目写进简历。

## 当前项目推荐的 Skills

### 1. 粒子交互系统

适用任务：

- 做 Canvas 粒子
- 做 Three.js 粒子
- 做鼠标吸引、点击爆炸、文字粒子、星云效果

要求：

- 必须先做可运行版本。
- 粒子数量要考虑性能。
- 要有鼠标或触摸交互。
- 最后要总结项目亮点。

### 2. 作品集网站

适用任务：

- 个人主页
- 项目展示页
- 实习求职作品页

要求：

- 第一屏要直接展示核心内容。
- 不要做空洞的营销页。
- 项目卡片要能说明技术栈和亮点。
- 页面要适配手机和电脑。

### 3. 新手教学

适用任务：

- 解释代码
- 拆解项目
- 制定学习计划

要求：

- 每一步说清楚原因。
- 不要默认用户懂框架、构建工具、终端命令。
- 给出可复制执行的命令。
- 解释常见报错是什么意思。

### 4. 简历项目包装

适用任务：

- 把项目写成简历内容
- 准备面试讲法
- 生成 GitHub README

要求：

- 表达真实，不夸大。
- 强调可展示成果、技术点、个人贡献。
- 给出面试时可以怎么讲。
